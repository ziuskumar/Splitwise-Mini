## Validation Rules Applied

The CSV import pipeline performs the following validations during ingestion:

* Date format validation
* Currency normalization
* Duplicate detection
* Negative amount handling
* Split consistency verification
* Participant activity validation
* Required field checks
* Invalid or malformed row detection

---

## Automated Actions Performed

The system supports the following automated correction mechanisms:

* Automatic refund categorization for negative amounts
* Currency conversion using configured exchange rates
* Duplicate transaction prevention
* Invalid row rejection with error logging
* Review flagging for suspicious or inconsistent records

---

## Accessing the Report in the Application

1. Open the application dashboard
2. Navigate to **Import → Report**
3. Upload a CSV file
4. Review detected anomalies and system actions in the generated report view

The UI provides expandable row-level details for debugging, correction tracking, and audit history.
